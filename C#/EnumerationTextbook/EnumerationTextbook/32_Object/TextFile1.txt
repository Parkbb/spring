// 개체(Object, 오브젝트): 설계도인 클래스로부터 만들어진 하나의 인스턴스(실체, 구성 요소)

개체 생성 순서
Step 1: 메모리 할당
힙(heap)에 메모리를 할당하기 위해서는 new 키워드를 사용

Step 2: 생성자를 통한 개체 초기화
클래스의 이름과 괄호 기호를 사용

Object LifeCycle

개체 생성 시기
- new 연산자를 이용한 메모리를 할당하고 개체를 생성
- 생성자를 이용해 메모리 상의 개체 초기화

개체 사용 시기
- 개체에 대한 메서드, 멤버 호출 / 접근

개체 소멸 시기
- 개체에게 할당된 메모리는 실행 환경으로 회수
- 메모리 해제
- 소멸 시기가 비결정적임
- GC가 개체 소멸을 관장

클래스 라이프사이클
-할당(Allocation)
-- Managed heap 메모리 공간에 CLR에 의해서 개체를 위한 메모리 할당

-인스턴스화(Instantiation)
-- 클래스의 구조에 맞게 메모리 초기화
-- 클래스의 생성자(Constructor)가 호출됨

-해제(Deallocation)
-- 가비지 수집기가 사용되지 않는 개체를 소멸
-- 경우에 따라서넌 IDisposable 인터페이스를 구현하여 Dispose 메서드를 구현할 수 있음

정적(스태틱) 멤버와 인스턴스 멤버
- 정적 멤버 : static 키워드가 붙은 멤버
-- 정적 필드
-- 정적 메서드
-- 접근 : 클래스명.멤버명 Math 클래스

- 인스턴스 멤버
-- 클래스내에서 static 키워드가 붙지않은 멤버
-- 클래스의 인스턴스 생성 후 멤버에 접근
-- static 키워드 제거
-- 접근 : 클래스 개체 = new 클래스(); 개체.멤버명
